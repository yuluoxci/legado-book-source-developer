# Skill 文档优化总结

## 优化概述

本次优化重点强化了以下方面：

1. ✅ **必须获取分析真实源码** - 禁止基于假设或推测
2. ✅ **JavaScript代码分析** - 必须从JS中提取真实的搜索模式和接口内容
3. ✅ **用户交互机制** - 不确定时必须询问用户并提供建议
4. ✅ **Python工具优化** - 使开发速度更快捷更准确

## 1. 核心原则强化

### 绝对禁止

- ❌ **禁止基于假设进行分析**
  - 不允许猜测HTML结构
  - 不允许推断CSS选择器
  - 不允许推测JavaScript逻辑

- ❌ **禁止不获取真实HTML源码就编写规则**
  - 必须获取并保存真实HTML
  - 必须基于真实HTML编写选择器
  - 必须验证每个选择器

- ❌ **禁止不分析JavaScript就推断搜索接口参数**
  - 必须下载并分析JS文件
  - 必须提取真实的API端点
  - 必须理解参数生成逻辑

- ❌ **禁止不确定时继续下一步**
  - 遇到不确定的情况必须停下来
  - 必须询问用户
  - 必须等待用户回复后继续

### 必须做到

- ✅ **必须获取并分析真实HTML源码**
  - 使用浏览器开发者工具
  - 使用Python工具获取
  - 永久保存供后续分析

- ✅ **必须分析网站的JavaScript代码**
  - 下载所有相关JS文件
  - 提取搜索相关函数
  - 找到API端点和参数生成逻辑

- ✅ **必须通过浏览器Network面板验证真实请求**
  - 复制cURL命令
  - 分析请求头和参数
  - 验证响应内容

- ✅ **不确定时必须询问用户并提供建议**
  - 提供清晰的问题描述
  - 提供多个解决方案
  - 等待用户回复后继续

## 3. 新增的Python工具

### tools/quick_analyze.py（全新工具）

**功能：**
快速网站分析工具，自动完成多个分析任务

**核心功能：**
1. 检测网站编码
2. 查找并下载JavaScript文件
3. 分析JavaScript内容
4. 搜索可能的搜索接口模式
5. 分析HTML结构
6. 测试搜索接口
7. 生成JSON报告

**使用方法：**
```bash
cd tools
python quick_analyze.py https://www.example.com
```

**输出：**
- home.html - 首页HTML
- search_result_*.html - 搜索结果HTML
- *.js - 下载的JavaScript文件
- analysis_report.json - 详细分析报告

**自动化分析：**
- 查找书籍列表容器
- 查找书籍信息元素
- 测试多个搜索URL模式
- 保存所有测试结果

**优势：**
- 一次性完成多个分析任务
- 自动保存所有相关文件
- 生成结构化的JSON报告
- 大大提高分析速度

### tools/js_param_analyzer.py（全新工具）

**功能：**
JavaScript参数分析工具，提取参数生成逻辑

**核心功能：**
1. 从HTML提取所有JavaScript
2. 下载并分析外部JS文件
3. 查找搜索相关函数
4. 查找API端点
5. 查找参数生成函数
6. 查找加密库引用
7. 分析cURL命令

**使用方法：**
```bash
# 分析网站
cd tools
python js_param_analyzer.py --analyze https://www.example.com

# 分析cURL命令
python js_param_analyzer.py --curl 'curl http://example.com...'
```

**输出：**
- *.js - 下载的JavaScript文件
- js_param_report.json - 参数分析报告

**分析内容：**
- 搜索相关函数列表
- API端点列表
- 参数生成函数列表
- 加密库使用情况
- cURL请求解析

**优势：**
- 自动提取关键函数
- 识别可能的API端点
- 查找参数生成逻辑
- 支持cURL命令分析

## 5. 使用示例

### 示例1：分析新网站

```bash
# 1. 使用浏览器开发者工具获取cURL
# (手动操作，F12 -> Network -> 复制cURL)

# 2. 分析cURL
cd tools
python js_param_analyzer.py --curl '复制的cURL命令'

# 3. 快速分析网站
python quick_analyze.py https://www.example.com

# 4. 查看分析报告
cat analysis_report.json
cat js_param_report.json

# 5. 检查保存的HTML文件
cat home.html
cat search_result_1.html

# 6. 基于真实源码编写规则
# (遵循WORKFLOW.md的后续步骤）
```

### 示例2：遇到不确定的情况

```
# 假设无法确定搜索接口的参数生成方式

1. 查阅用户交互指南
   cat references/用户交互指南.md

2. 找到对应场景
   场景1：搜索接口参数来源不明

3. 按照格式询问用户
   【问题】搜索接口需要以下参数，但不确定它们的生成方式：
   
   参数列表：
   - timestamp: 时间戳
   - signature: 签名（需要查看JS代码）
   - token: 令牌（可能需要登录）
   
   【建议】
   1. 请打开浏览器开发者工具（F12）
   2. 切换到 Sources 标签
   3. 搜索关键词：signature、token、encrypt、sign
   4. 找到相关的函数定义
   
   请用户提供：
   - 登录账号（如果有）
   - JavaScript文件中相关的代码片段
   - 或者网站是否有登录要求

4. 等待用户回复

5. 根据用户回复继续分析
```

## 10. 总结

本次优化的核心思想是：

**基于真实源码分析，严格遵循工作流程，不确定时主动交互，使用自动化工具提高效率。**

通过以下措施实现：

1. ✅ 强化核心原则（禁止猜测，必须真实分析）
2. ✅ 新增用户交互机制（8个场景+标准模板）
3. ✅ 开发自动化工具（quick_analyze.py + js_param_analyzer.py）
4. ✅ 优化文档结构（SKILL.md + WORKFLOW.md + 用户交互指南.md）
5. ✅ 建立清晰的工作流程（明确步骤+决策点+检查清单）

最终目标是：
- 提高分析速度（自动化工具）
- 提高准确性（真实源码）
- 改善用户体验（交互机制）
- 降低错误率（严格流程）
