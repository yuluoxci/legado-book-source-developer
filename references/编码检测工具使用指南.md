# 编码检测工具使用指南

## 概述

Legado 书源智能体现在具备**智能编码检测功能**，可以自动识别网站的编码类型（UTF-8、GBK、GB2312等），帮助您快速配置书源规则，避免编码问题导致的乱码。

## 工具列表

智能体提供两个编码检测工具：

### 1. detect_charset
**功能**：通过 URL 检测网站的编码类型

**用法**：
```
检测 https://www.example.com 的编码
```

**返回结果**：
```json
{
  "url": "https://www.example.com",
  "charset": "gbk",
  "confidence": 0.95,
  "source": "headers",
  "details": {
    "headers": "gbk",
    "meta": "gbk",
    "content": {
      "charset": "gbk",
      "confidence": 0.99
    },
    "characters": {
      "charset": "gbk",
      "confidence": 0.85
    }
  },
  "recommendation": {
    "charset_config": "\"gbk\"",
    "usage_example": "/search,{\"charset\":\"gbk\"}"
  }
}
```

### 2. detect_charset_from_html
**功能**：从 HTML 内容中检测编码类型

**用法**：
```
检测这段 HTML 的编码：<html><head><meta charset="gbk"></head>...
```

**返回结果**：
```json
{
  "charset": "gbk",
  "confidence": 0.90,

  "details": {
    "meta": "gbk",

  }
}
```

## 检测方法

编码检测工具使用多种方法来确保准确性：

### 1. HTTP 响应头检测
- 检查 `Content-Type` 头中的 `charset` 参数
- 优先级：高
- 可靠性：高

### 2. HTML meta 标签检测
- 检查 `<meta charset="...">` 标签
- 检查 `<meta http-equiv="Content-Type" content="...">` 标签
- 优先级：高
- 可靠性：中

### 3. 内容分析（chardet）
- 使用 chardet 库分析页面内容
- 统计字符频率推断编码
- 优先级：中
- 可靠性：中

### 4. 中文字符特征检测
- 尝试用不同编码解码
- 统计成功解码的中文字符数量
- 优先级：低
- 可靠性：低（仅作参考）

## 使用场景

### 场景 1：开始开发书源前
在开始分析网站结构之前，先检测网站的编码：

```
请检测 https://www.69shuba.com 的编码类型
```

智能体会自动：
1. 访问网站
2. 检测编码
3. 返回编码配置建议

### 场景 2：遇到乱码问题时
如果书源显示乱码，让智能体检测编码：

```
我的书源 https://example.com 显示乱码，请检测一下编码
```

智能体会：
1. 检测网站的正确编码
2. 对比当前书源配置
3. 给出修复建议

### 场景 3：配置 searchUrl 时
在配置搜索 URL 时，让智能体自动添加编码配置：

```
帮我为 https://www.example.com 配置 searchUrl
```

智能体会自动：
1. 检测网站编码
2. 在 searchUrl 中添加 `"charset":"gbk"`
3. 提供完整的配置

## 编码标准化

工具会自动将各种编码名称标准化：

| 原始编码 | 标准化后 | 说明 |
|---------|---------|------|
| GBK, gbk, x-gbk | gbk | 简体中文 |
| GB2312, gb2312 | gbk | 简体中文（GBK 子集） |
| GB18030 | gbk | 完整中文标准 |
| UTF-8, utf8, utf-8 | utf-8 | 国际标准 |
| BIG5 | gbk | 繁体中文（归类到 GBK） |

## 配置建议

工具会自动生成配置建议：

### 简单配置
```json
{
  "searchUrl": "/search,{\"charset\":\"gbk\"}"
}
```

### 高级配置（JavaScript）
```javascript
@js:
var option = {
  "charset": "gbk",
  "method": "POST",
  "body": String(body)
};
"https://www.example.com/search," + JSON.stringify(option)
```

## 最佳实践

### 1. 开发流程
```
1. 检测网站编码
   ↓
2. 分析网站结构
   ↓
3. 配置书源规则
   ↓
4. 添加 charset 配置（如果不是 UTF-8）
   ↓
5. 测试验证
```

### 2. 检查清单
- [ ] 开发前检测网站编码
- [ ] 如果是 GBK，在所有请求中添加 `"charset":"gbk"`
- [ ] POST 请求的 body 用 `String()` 包裹
- [ ] 使用 `java.encodeURI(key, 'GBK')` 编码 URL 参数
- [ ] 用中文关键字测试，确保无乱码

### 3. 常见问题

**Q: 检测结果不确定怎么办？**
A: 如果置信度较低（< 0.7），建议手动检查：
1. 打开浏览器开发者工具（F12）
2. 切换到 Network 标签
3. 查看 Response Headers 中的 Content-Type

**Q: GB2312 和 GBK 有什么区别？**
A: GB2312 是 GBK 的子集，工具会自动将 GB2312 标准化为 GBK，两者在书源中可以通用。

**Q: UTF-8 需要指定 charset 吗？**
A: 不需要。UTF-8 是默认编码，可以省略 `"charset":"utf-8"`。

## 技术细节

### 检测优先级
```
1. HTTP 响应头（最高）
   ↓
2. HTML meta 标签
   ↓
3. 内容分析（chardet）
   ↓
4. 中文字符特征（最低）
```

### 置信度说明
- **0.95 - 1.00**: 非常可靠，可以直接使用
- **0.80 - 0.95**: 可靠，建议验证
- **0.70 - 0.80**: 可用，建议手动检查
- **< 0.70**: 不确定，建议手动检测

### 错误处理
如果检测失败，工具会返回：
```json
{
  "charset": "utf-8",
  "confidence": 0.0,
  "source": "fallback",
  "errors": ["检测失败: 请求超时"]
}
```

## 总结

编码检测工具可以大大简化书源开发流程：

✅ **自动检测** - 无需手动检查响应头
✅ **智能分析** - 多种方法确保准确性
✅ **配置建议** - 自动生成配置代码
✅ **标准化处理** - 统一编码名称

**推荐用法**：在开发任何书源之前，先让智能体检测网站编码！
