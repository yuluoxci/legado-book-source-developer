## 🎨 常见正则表达式模式

### 清理前缀/后缀

```javascript
// 删除前缀
"作者：张三" ##^作者：## → "张三"
"《书名》" ##^《|》$## → "书名"

// 删除后缀
"分类：玄幻" ##^.*分类：## → "玄幻"
```

### 提取特定内容

```javascript
// 提取"|"前面的内容
"玄幻 | 作者：张三" ##^[^|]*## → "玄幻 "

// 提取"|"后面的内容
"玄幻 | 作者：张三" ##.*\|## → " 作者：张三"

// 提取数字
"字数：100万字" ##[^0-9]## → "100"

// 多重替换
"《书名》-作者-分类" ##^《|》|^-作者-|^-## → "书名"
```

### 格式化输出

```javascript
// 组合多个字段
"2026-01-06" ##(\d{4})-(\d{2})-(\d{2})##$1年$2月$3日## → "2026年01月06日"

// 清理空白
"  内容  " ##\s+## → "内容"
```

## ⚠️ 常见陷阱和注意事项

### 1. 选择器误用

❌ **错误**：
```js
{
  "name": "@css:.title@text"  // 滥用@css前缀
}
```

✅ **正确**：
```js
{
  "name": ".title@text"  // 直接使用CSS选择器
}
```

### 2. 提取类型混淆

❌ **错误**：
```js
{
  "content": "#content@text"  // 正文应该用@html
}
```

✅ **正确**：
```js
{
  "content": "#content@html"
}
```

### 3. 正则表达式错误

❌ **错误**：
```js
{
  "author": ".author@text##^作者"  // 缺少结束符
}
```

✅ **正确**：
```js
{
  "author": ".author@text##^作者：##"
}
```

### 4. 未处理特殊情况

❌ **错误**：
```js
{
  "coverUrl": "img@src"  // 未考虑懒加载
}
```

✅ **正确**：
```js
{
  "coverUrl": "img@data-original||img@src"  // 兼容懒加载
}
```

## 📚 学习要点总结

### 必须掌握的技能

1. **CSS选择器**：熟练掌握类选择器、ID选择器、标签选择器、复合选择器
2. **提取类型**：理解@text、@html、@href、@src等提取类型的区别
3. **正则表达式**：能够使用##进行文本清理和格式化
4. **JavaScript处理**：了解<js>标签的基本用法（用于复杂逻辑）

### 最佳实践

1. **优先使用Default语法**：`class.content@text` 比 `@css:.content@text` 更简洁
2. **验证选择器**：必须在真实HTML上验证选择器有效性
3. **处理边界情况**：考虑空值、懒加载、信息合并等情况
4. **保持简洁**：避免过度使用复杂的JavaScript和XPath
