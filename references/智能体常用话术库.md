#### 用户查询@text和@html的区别
```
📚 @text vs @html 对比

【@text - 提取纯文本】
✅ 用途：提取标题、作者、正文等纯文本
✅ 示例：.content@text → "正文内容"
❌ 特点：自动去除HTML标签

【@html - 提取完整结构】
✅ 用途：需要保留格式、调试页面结构
✅ 示例：.content@html → "<p>正文内容</p>"
❌ 特点：保留所有标签

【选择建议】
- 90%的正文提取：使用 @text
- 保留段落格式：使用 @html 或 @textNode
- 调试页面：使用 @html

```

#### 用户查询正则表达式
```
📚 正则表达式使用指南

【基本格式】
选择器@提取类型##正则表达式##替换内容

【常见场景】

1️⃣ 清理前缀
作者：.author@text##^作者：##
结果："作者：张三" → "张三"

2️⃣ 清理后缀
时间：.time@text##更新时间：$##
结果："更新时间：2024-01-18" → "2024-01-18"

3️⃣ 提取中间内容
信息：.info@text##.*作者：(.*?)##$1
结果："科幻 | 作者：张三" → "张三"

4️⃣ 替换内容
内容：.content@text##\n##<br>
结果：将换行符替换为<br>标签

【技巧】
- ^匹配开头，$匹配结尾
- .*?非贪婪匹配
- (.*?)捕获分组
- |表示或

有具体需要处理的文本？发给我帮你写正则！
```

### 1.2 POST请求配置相关

#### 用户查询POST请求
```
📚 POST请求配置指南

【基本格式】
https://www.example.com/search,{"method":"POST","body":"keyword={{key}}&page={{page}}","charset":"gbk"}

【重要规则】✅ 必须遵守
1. body必须用String()包裹
2. charset为utf-8时可省略
3. 无特殊情况不需要请求头和webView

【示例1：简单POST】
```
https://www.example.com/search,{"method":"POST","body":"searchkey={{key}}&page={{page}}"}
```

【示例2：复杂POST（使用JavaScript）】
```
@js:
var headers = {"User-Agent": "Mozilla/5.0..."};
var body = "keyword=" + String(key) + "&page=" + String(page);
var option = {"method":"POST","body":String(body),"headers":headers,"charset":"gbk"};
"https://www.example.com/search," + JSON.stringify(option)
```

【常见错误】❌
- ❌ body不用String()包裹
- ❌ 编造请求头和参数
- ❌ 不使用JSON.stringify()

需要配置具体的网站POST请求？提供网站URL！
```
