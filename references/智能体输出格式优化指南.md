## 书源JSON格式

### 2.1 完整书源JSON

#### 格式要求
- 使用代码块包裹
- 添加注释说明
- 标注可复制区域

#### 模板
```
🎉 书源创建成功！

【书源信息】
- 书源名称：{书源名称}
- 书源地址：{书源地址}
- 书源类型：{小说/漫画/有声书}

【完整JSON】（可直接复制导入）

```json
[
  {
    "bookSourceName": "书源名称",
    "bookSourceUrl": "https://example.com",
    "bookSourceType": 0,
    "searchUrl": "/search?q={{key}}",
    "ruleSearch": {
      "bookList": ".book-list .item",
      "name": ".title@text",
      "author": ".author@text",
      "bookUrl": "a@href"
    },
    "ruleBookInfo": {
      "name": "h1@text",
      "author": ".author@text",
      "tocUrl": ""
    },
    "ruleToc": {
      "chapterList": "#list a",
      "chapterName": "text",
      "chapterUrl": "href"
    },
    "ruleContent": {
      "content": "#content@text",
      "nextContentUrl": ""
    }
  }
]
```

【使用方法】
1. 复制上面的JSON
2. 打开Legado阅读APP
3. 进入 书源管理 → 导入书源
4. 粘贴JSON并确认

【注意事项】
✅ 所有规则已基于真实HTML验证
✅ POST请求配置符合规范
✅ 特殊情况已处理

【复制提示】
📋 点击代码块右上角的"复制"按钮，或手动选中复制
```

### 2.2 单个规则片段

#### 格式要求
- 使用inline code或代码块
- 添加前后对比

#### 模板
```
【规则修改】

修改前：
```json
"coverUrl": ""
```

修改后：
```json
"coverUrl": "img@src||img@data-original"
```

【说明】
- 添加了备选选择器处理懒加载图片
- 优先使用data-original，失败则使用src
```

## 代码块格式

### 3.1 JavaScript代码

#### 格式要求
- 添加注释说明
- 标注变量含义

#### 模板
```
【JavaScript代码示例】

```javascript
// POST请求配置示例
var headers = {
  "User-Agent": "Mozilla/5.0..."
};
var body = "keyword=" + String(key) + "&page=" + String(page);
var option = {
  "charset": "gbk",
  "method": "POST",
  "body": String(body),
  "headers": headers
};
"https://www.example.com/search," + JSON.stringify(option)
```

【代码说明】
- headers：请求头配置
- body：请求体（必须用String()包裹）
- option：完整的请求配置
- 最后拼接URL和配置

【使用场景】
- 复杂的POST请求
- 需要动态生成请求头
- 需要处理特殊字符
```

### 3.2 CSS选择器代码

#### 格式要求
- 添加前后对比
- 标注选择器类型

#### 模板
```
【CSS选择器示例】

```css
/* 提取书名 */
.book-title@text

/* 提取作者 */
.author@text##^作者：##

/* 提取封面（处理懒加载） */
img.lazy@data-original||img@src

/* 提取链接 */
a@href

/* 提取HTML结构 */
.content@html##<script[\s\S]*?</script>
```

【选择器说明】
- @text：提取纯文本
- @html：提取完整HTML
- @href/@src：提取属性
- ||：备选选择器
- ##：正则表达式
```

## 表格格式

### 5.1 对比表格

#### 格式要求
- 清晰的对比
- 添加状态标记

#### 模板
```
【@text vs @html 对比】

| 特性 | @text | @html |
|------|-------|-------|
| 提取内容 | 纯文本 | 完整HTML |
| 去除标签 | ✅ 自动去除 | ❌ 保留标签 |
| 保留格式 | ❌ 不保留 | ✅ 保留格式 |
| 使用场景 | 标题、作者、正文 | 调试、需要格式 |

【选择建议】
- 90%的正文提取：使用 @text
- 保留段落格式：使用 @html 或 @textNode
- 调试页面结构：使用 @html
```

### 5.2 参数说明表格

#### 格式要求
- 详细的参数说明
- 添加示例

#### 模板
```
【POST请求参数说明】

| 参数 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| method | String | ❌ | 请求方法 | "POST" |
| body | String | ✅ | 请求体 | "keyword=test" |
| charset | String | ❌ | 编码格式 | "gbk" |
| headers | Object | ❌ | 请求头 | {"User-Agent":"..."} |
| webView | Boolean | ❌ | 是否使用WebView | true |

【注意事项】
- body必须用String()包裹
- charset为utf-8时可省略
- 无特殊情况不需要headers和webView
```
