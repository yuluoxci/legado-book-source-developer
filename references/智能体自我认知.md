## 🚨 重要规则和约束

### 必须遵守的规则

#### 1. 输出格式规范

**必须严格按照Legado书源JSON标准格式输出（数组格式）**

✅ **正确格式**:
```json
[
  {
    "bookSourceUrl": "https://example.com",
    "bookSourceName": "XX小说网",
    "bookSourceType": 0,
    "searchUrl": "https://example.com/search?q={{key}}",
    "ruleSearch": {
      "bookList": "ul.book-list li",
      "name": "h2@text",
      "author": "span.author@text",
      "bookUrl": "a@href"
    }
  }
]
```

❌ **错误格式**:
```json
{
  "bookSourceUrl": "https://example.com",
  "bookSourceName": "XX小说网"
}
```

#### 2. 严禁使用的字段

**1. 严禁使用`prevContentUrl`字段**（Legado中不存在）

❌ **错误**:
```json
"ruleContent": {
  "content": ".content@text",
  "prevContentUrl": ".prev@href"  // ❌ 此字段不存在！
}
```

✅ **正确**:
```json
"ruleContent": {
  "content": ".content@text",
  "nextContentUrl": ".next@href"  // ✅ 只有nextContentUrl
}
```

**2. 严禁直接提取`<select>`元素的value**（select没有value属性）⭐ 重要

当使用select下拉菜单实现分页时，必须提取`<option>`的value，而不是`<select>`的value。

❌ **错误**:
```json
"ruleToc": {
  "nextTocUrl": "select@value"  // ❌ select没有value属性
}
```

✅ **正确**:
```json
"ruleToc": {
  "nextTocUrl": "select option@value"  // ✅ 提取option的value
}
```

✅ **最佳实践**:
```json
"ruleToc": {
  "nextTocUrl": "select[name='pageselect'] option:not([selected])@value"  // ✅ 精确定位并排除已选中
}
```

#### 3. 严禁使用的CSS伪类选择器

**以下伪类选择器在Legado中不可用**：

❌ **错误**:
```css
:a:contains("下一章")   // ❌ :a:contains() 不可用
:first-child           // ❌ :first-child 不可用
:last-child            // ❌ :last-child 不可用
```

✅ **正确**:
```css
text.下一章            // ✅ 使用 text.文本 格式
.0                     // ✅ 使用 .0 表示第一个元素
.-1                    // ✅ 使用 .-1 表示最后一个元素
```

#### 4. 正则表达式使用规范

**必须使用`##`分隔符**

✅ **删除广告**:
```css
.content@text##广告##
```

✅ **替换文本**:
```css
.content@text##(\d{4})-(\d{2})-(\d{2})##$1年$2月$3日##
```

✅ **提取内容**:
```css
.content@text##(\d+)##$1##
```

**捕获组用法**:
- 使用`()`包裹需要捕获的部分
- 使用`$1`, `$2`, `$3`引用捕获组

#### 5. nextContentUrl 判断规则

**区分"下一章"和"下一页"**:

✅ **应该设置nextContentUrl**:
- "下一章"
- "下章"
- "下一节"
- "下一回"

❌ **不应该设置nextContentUrl**:
- "下一页"
- "下一页阅读"
- "继续阅读"
- "下一页（1/10）"

#### 6. nextTocUrl 判断规则（目录页分页）

**重要**: 当目录页有多个分页时，必须设置`nextTocUrl`！

**使用场景**:
- 目录页显示"第1页"、"第2页"、"第3页"等分页
- 目录页有"下一页"、"末页"等按钮
- 章节列表被分为多个页面显示

**支持格式**:

**格式1: 单个URL（CSS选择器或XPath）**
```json
"ruleToc": {
  "chapterList": "//div[@class='list']//li",
  "chapterName": "//a/text()",
  "chapterUrl": "//a/@href",
  "nextTocUrl": "//div[@class='pagination']//li[last()-1]/a/@href"
}
```

**格式2: URL数组（使用JS脚本动态生成）**
```json
"ruleToc": {
  "chapterList": "//div[@class='list']//li",
  "chapterName": "//a/text()",
  "chapterUrl": "//a/@href",
  "nextTocUrl": "@js:\nconst pages = [];\nlet end = java.getString('#end@href');\nlet num = +end.match(/(\\d+)\\.html/)[1] || 0;\nfor (let i = 2; i <= num; i++) {\n  pages.push(book.tocUrl + i + '.html');\n}\npages"
}
```

**格式3: select下拉菜单分页（CSS选择器）** ⭐ 重要发现

当网站使用`<select>`下拉菜单实现分页时：

**HTML结构示例**:
```html
<select name="pageselect" onchange="self.location.href=options[selectedIndex].value">
  <option value="/biquge_317279/1/#all" selected="selected">1 - 30章</option>
  <option value="/biquge_317279/2/#all">31 - 60章</option>
  <option value="/biquge_317279/3/#all">61 - 90章</option>
</select>
```

**规则写法**:
```json
"ruleToc": {
  "chapterList": "//div[@id='list']//li",
  "chapterName": "//a/text()",
  "chapterUrl": "//a/@href",
  "nextTocUrl": "select[name='pageselect'] option:not([selected])@value"
}
```

**关键要点**:
- ✅ **正确**: `select option@value` - 提取`<option>`元素的`value`属性
- ❌ **错误**: `select@value` - `<select>`元素本身没有`value`属性
- ✅ **最佳**: `select option:not([selected])@value` - 排除已选中的选项，避免循环

**选择器说明**:
- `select[name='pageselect']` - 定位name为pageselect的下拉菜单
- `option:not([selected])` - 选择未选中的选项
- `@value` - 提取选项的value属性（分页URL）

**高级用法**: 使用JS脚本生成所有分页URL
```json
"nextTocUrl": "@js:\nvar select = document.querySelector('select[name=\"pageselect\"]');\nvar options = select.options;\nvar urls = [];\nfor (var i = 0; i < options.length; i++) {\n  if (!options[i].selected) {\n    urls.push(options[i].value);\n  }\n}\nurls"
```

**格式4: 不需要分页（留空）**
```json
"ruleToc": {
  "chapterList": "//div[@class='list']//li",
  "chapterName": "//a/text()",
  "chapterUrl": "//a/@href",
  "nextTocUrl": ""
}
```

**停止条件**:
当以下情况时，停止加载下一页：
- 返回空数组`[]`
- 返回`null`
- 返回空字符串`""`

**判断示例**:

✅ **需要设置nextTocUrl**:
- 目录页底部显示"首页 1 2 3 ... 10 末页"
- 有"下一页"按钮指向新的章节列表
- URL格式如`/catalog/1.html`, `/catalog/2.html`等

❌ **不需要设置nextTocUrl**:
- 目录页只有一页
- 所有章节都在一个页面显示
- 没有分页按钮

**与nextContentUrl的区别**:

| 字段 | 位置 | 用途 | 示例 |
|------|------|------|------|
| **nextTocUrl** | `ruleToc` | 目录页分页 | 获取更多章节列表 |
| **nextContentUrl** | `ruleContent` | 正文页分页 | 同一章节的更多内容 |

#### 7. 必须基于真实HTML 🚨⭐⭐⭐⭐⭐

**严禁编造规则！必须使用最真实的HTML！**

### 🚨 为什么要使用最真实的HTML？

**真实HTML**和**浏览器开发者工具看到的HTML**可能存在巨大差异：

#### 1. JavaScript动态注入的内容
```
浏览器开发者工具（美化后）:
<div class="book-info">
  <h1 class="title">书名</h1>
</div>

真实HTTP请求返回的HTML（原始）:
<div class="book-info" id="bookInfo" data-id="12345">
  <!-- JS注入的内容在这里 -->
</div>
```

#### 2. 注释节点和隐藏元素
```
真实HTML可能包含:
- <!-- 广告位占位 -->
- <div class="hidden" style="display:none">隐藏内容</div>
- <script type="text/template">模板内容</script>
```

#### 3. 空白字符和格式
```
浏览器开发者工具（自动美化）:
<div>
  <span>内容</span>
</div>

真实HTML（原始，可能压缩）:
<div><span>内容</span></div>
```

#### 4. class和ID名称的差异
```
开发者工具显示:
<div class="book-info">

实际HTML可能是:
<div class="book-info    "  // 有多个空格
<div class="book-info active"  // 动态添加的class
<div class="book-info_20250120"  // 带有版本号
```

### ✅ 正确流程

```
1. 调用 `smart_fetch_html()` 获取真实的HTTP响应HTML
   - 完整的HTML源代码
   - 包含所有注释、空格、隐藏元素
   - 原始格式，未经过浏览器美化

2. 调用 `analyze_complete_book_source()` 分析真实HTML
   - 使用BeautifulSoup解析真实HTML
   - 识别真实的class、ID、结构
   - 考虑动态加载的内容

3. 基于真实HTML编写规则
   - 选择器必须匹配真实的class/ID
   - 考虑HTML的实际情况
   - 处理可能的变体（如动态class）

4. 调用 `debug_book_source()` 验证规则
   - 使用真实HTML测试规则
   - 确保规则在真实环境下可用
```

### ❌ 错误流程

```
1. 仅查看浏览器开发者工具的HTML
   ❌ 开发者工具可能美化HTML
   ❌ 可能显示JS动态注入的内容
   ❌ 可能隐藏重要元素

2. 猜测HTML结构
   ❌ 假设存在某个class
   ❌ 假设某个元素的位置
   ❌ 基于其他网站的经验

3. 生成编造的规则
   ❌ 选择器不匹配真实HTML
   ❌ 规则基于假设而非事实
   ❌ 书源无法正常工作
```

### ⚠️ 常见陷阱

#### 陷阱1: 开发者工具显示的class和实际不同
```
开发者工具: <div class="book-title">
实际HTML:    <div class="book-title main-title current">
解决: 必须使用实际HTML中的完整class名称
```

#### 陷阱2: JS动态加载的内容
```
开发者工具显示: <div class="content">正文内容</div>
实际HTTP返回:   <div class="content" id="content">加载中...</div>
解决: 需要启用webView或查找真实的API接口
```

#### 陷阱3: 注释节点
```
实际HTML:
<div class="book-info">
  <!-- <h1 class="title">旧标题</h1> -->
  <h1 class="title-new">新标题</h1>
</div>
解决: 确保选择器匹配的是实际生效的元素
```

| **格式** | 原始/可能压缩 | 美化格式化 |

### 黄金技巧（来自精华文档）

#### 1. CSS选择器简写

**完整写法** → **简写**
```
class.名字1@text          →  .名字1@text
class.名字1 名字2 名字3@text  →  .名字1.名字2.名字3@text
class.xxx@li@a@text      →  .xxx li a@text
id.最优选哦@text          →  #最优选哦@text
```

**原则**:
- 优先使用id（唯一性强）
- 同一标签里有id和class时，优先选id
- 多个class中，如果某个唯一出现，可以单独使用
- `@`可用空格代替（多层选择时）

#### 2. 解决搜索30秒超时

**方法1: 登录检查JS**
```javascript
cookie.removeCookie(source.getKey())
result
```

**方法2: 搜索地址清空cookie**
```
{{cookie.removeCookie(source.getKey())}}/search.html?key={{key}}
```

#### 3. 请求头配置（防盗链）

**基本配置**：
```json
{
  "User-Agent": "Mozilla/5.0 (Linux; Android 11) Mobile Safari/537.36"
}
```

**防盗链处理**：
```json
{
  "User-Agent": "Mozilla/5.0 (Linux; Android 11) Mobile Safari/537.36",
  "Referer": "{{baseUrl}}"
}
```

**动态Referer**：
```json
{
  "headers": {
    "Referer": "{{baseUrl}}"
  }
}
```

#### 4. 去广告技巧

**使用@ownText代替@text**:
```
.content@ownText  // 只取当前标签文字，排除子标签广告
```

**正则净化**:
```
.content@text##广告词##
```

#### 5. 章节倒序

**列表规则前加`-`号**:
```
-ul.chapter-list li  // 最新章在前时，倒序显示
```

### 规则口诀

```
列表定位不加@，字段提取必带@。
位置默认就是0，不写也行。
text全都要，ownText最干净。
html带标签，属性加@不能忘。
净化放最后，网站千变要灵活。
```

### 提取类型速查表

| 类型 | 作用 | 示例 | 记忆 |
|------|------|------|------|
| `@text` | 取所有文字（包括子标签） | `h2@text` | 全都要 |
| `@ownText` | 只取当前标签文字，不要子标签 | `p@ownText` | 最干净 |
| `@html` | 取完整HTML（保留标签） | `div.content@html` | 带标签 |
| `@href` | 取链接地址 | `a@href` | 链接@ |
| `@src` | 取图片地址 | `img@src` | 图片@ |
| `@data-xxx` | 取自定义属性 | `div@data-id` | 自定义@ |

### 常见错误及避免

| 错误 | 原因 | 解决方案 |
|------|------|---------|
| 使用`prevContentUrl` | 不知道Legado不支持 | 使用`nextContentUrl` |
| 使用`:a:contains()` | 以为可用 | 使用`text.文本`格式 |
| 使用`:first-child` | 以为可用 | 使用`.0` |
| 正则表达式不用`##` | 不知道格式 | 使用`##`分隔符 |
| 编造规则 | 未获取真实HTML | 先调用`smart_fetch_html()` |
| 跳过知识库 | 节省时间 | 第一阶段必须查询知识库 |
