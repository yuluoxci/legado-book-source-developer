# 书源验证功能修复总结

## 问题描述

用户在上传书源时,如果书源内容格式不合规,工具会直接上传失败,导致用户体验不佳。

**需求:**
1. 当用户输入书源内容要求上传时,先检查书源内容是否合规
2. 如果不合规,尝试修正成正确格式后再上传
3. 提供清晰的验证反馈

### 2. 修改 BookSourceUploader 类

#### 2.1 集成验证流程

在 `upload_book_source` 方法开头添加验证:

```python
def upload_book_source(self, book_source_json, upload_config=None):
    # 步骤1: 验证书源内容并尝试修正
    is_valid, fixed_source, errors, warnings = self.validator.validate_and_fix(book_source_json)

    # 显示验证结果
    self._print_validation_result(is_valid, errors, warnings)

    # 如果验证失败,不执行上传
    if not is_valid:
        return False, {
            'error': '书源验证失败',
            'details': '书源内容不符合Legado标准格式',
            'errors': errors,
            'warnings': warnings
        }

    # 使用修正后的书源进行上传
    book_source_json = fixed_source
    # ... 继续上传流程
```

#### 2.2 添加验证结果显示

新增 `_print_validation_result` 方法,美化显示验证结果:

```python
def _print_validation_result(self, is_valid, errors, warnings):
    """打印验证结果"""
    print("\n" + "=" * 70)
    print("📋 书源验证结果")
    print("=" * 70)

    if is_valid:
        print("\n✅ 验证通过! 书源格式符合要求")
    else:
        print("\n❌ 验证失败! 书源格式存在问题")

    if warnings:
        print(f"\n⚠️  发现 {len(warnings)} 个警告:")
        for i, warning in enumerate(warnings, 1):
            print(f"  {i}. {warning}")

    if errors:
        print(f"\n🚫 发现 {len(errors)} 个错误:")
        for i, error in enumerate(errors, 1):
            print(f"  {i}. {error}")

    print("\n" + "=" * 70)
```
