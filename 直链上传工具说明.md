# 直链上传工具说明

## 概述

`upload_book_source.py` 是一个专门用于将Legado书源上传到图床/云存储的工具，生成可分享的直链。

**创建时间**: 2026-03-13
**版本**: v1.0

## 主要功能

✅ **一键上传书源到图床**
- 内置默认配置（鲸落图床）
- 支持自定义上传配置
- 自动提取下载链接

✅ **美化返回结果**
- 清晰的成功/失败提示
- 书源信息展示
- 上传信息展示
- 使用方法说明

✅ **灵活的配置支持**
- 支持多种图床服务
- 自定义请求头和请求体
- 支持JSONPath规则提取链接

## 使用方法

### 基本用法（使用默认配置）

```bash
cd tools
python upload_book_source.py book_source.json
```

### 使用自定义配置

```bash
python upload_book_source.py book_source.json upload_config.json
```

### 默认上传配置

```json
{
  "compress": false,
  "downloadUrlRule": "$.data.links.url",
  "summary": "鲸落图床",
  "uploadUrl": "https://tu.406np.xyz/api/v1/upload",
  "method": "POST",
  "headers": {
    "Accept": "application/json"
  },
  "body": {
    "file": "fileRequest",
    "show": "1"
  },
  "type": "multipart/form-data"
}
```

## 配置参数说明

| 参数 | 说明 | 示例 |
|------|------|------|
| `compress` | 是否压缩上传内容 | `true`/`false` |
| `downloadUrlRule` | 下载链接提取规则 | `"$..url"`（JSONPath） |
| `summary` | 图床服务名称 | `"鲸落图床"` |
| `uploadUrl` | 上传API地址 | `"https://tu.406np.xyz/api/v1/upload"` |
| `method` | 请求方法 | `"POST"` |
| `headers` | 请求头 | `{"Accept":"application/json"}` |
| `body` | 请求体配置 | `{"file":"fileRequest","show":"1"}` |
| `type` | 请求类型 | `"multipart/form-data"` 或 `"application/json"` |

**body 参数说明：**
- `fileRequest`：表示该字段是书源文件内容（固定值）
- 其他字段：根据图床API要求添加

## 下载链接提取规则

工具支持简化的JSONPath规则来提取下载链接：

| 规则 | 说明 | 示例 |
|------|------|------|
| `$.data.links.url` | 提取data.links.url字段 | `{"data": {"links": {"url": "..."}}}` |
| `$.data.url` | 提取data对象中的url字段 | `{"data": {"url": "..."}}` |
| `$..url` | 深度查找url字段 | `{"data": {"url": "..."}}` |
| `$` | 直接返回整个响应 | `{"url": "...", "expire": 3600}` |

## 技术细节

### BookSourceUploader类

**主要方法：**

1. `upload_book_source(book_source_json, upload_config)`
   - 上传书源到图床
   - 返回(success, result)元组

2. `_extract_download_url(data, rule)`
   - 根据JSONPath规则提取下载链接
   - 支持简化版的JSONPath语法

3. `_format_success_result(book_source, config, response, download_url)`
   - 格式化成功结果
   - 美化输出内容

### 错误处理

工具会处理以下错误情况：

- `requests.exceptions.Timeout` - 请求超时
- `requests.exceptions.ConnectionError` - 连接失败
- `json.JSONDecodeError` - 响应解析失败
- 其他异常情况

### 文件上传

- 使用 `requests.post()` 发送POST请求
- 支持 `multipart/form-data` 和 `application/json` 两种格式
- 书源文件名格式：`{bookSourceName}.json`
- 自动设置正确的Content-Type

## 注意事项

### 1. 隐私和安全

- ⚠️ 不要上传包含敏感信息的书源
- ⚠️ 使用自定义配置时，妥善保管API Token
- ⚠️ 定期检查上传的链接是否仍然有效

### 2. 图床服务

- 默认使用鲸落图床（免费服务）
- 可以自定义配置使用其他图床服务
- 不同图床的API接口可能不同，需要相应调整配置

### 3. 网络连接

- 需要稳定的网络连接
- 上传失败会显示详细的错误信息
- 建议在良好的网络环境下使用

### 4. 书源文件

- 确保书源文件是有效的JSON格式
- 建议先使用 `validate_book_source.py` 验证
- 文件大小不要超过图床限制

## 常见问题

### Q1: 上传失败怎么办？

A: 检查以下几点：
1. 网络连接是否正常
2. 图床服务是否可用
3. 上传配置是否正确
4. 书源文件格式是否正确

### Q2: 如何更换图床？

A: 创建自定义配置文件，修改以下参数：
- `uploadUrl`: 新的图床API地址
- `headers`: 新的请求头
- `body`: 根据新图床的API要求配置

### Q3: 上传的链接会失效吗？

A: 取决于图床服务：
- 部分图床有有效期限制
- 部分图床可能永久有效
- 建议定期检查链接是否可用

### Q4: 如何批量上传书源？

A: 使用shell脚本：
```bash
for file in book_source_*.json; do
    python upload_book_source.py "$file"
done
```
